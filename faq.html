Questions & Answers

FAQ

Q: So many megabytes for such a small plugin — what's in there, lead?
Nope, pure graphics power! 🚀 These plugins use Rust and the Iced framework with wgpu on board — a tiny portable graphics engine that lives inside the plugin. It brings its own cinema instead of relying on your operating system's wobbly folding chairs. The first 3–4 MB are reserved for the red carpet, lighting rig, and popcorn machine. Everything beyond that is actual audio magic. 🍿😎
Q: You call it "AI-first" — where's the artificial intelligence?
"AI-first" means the entire architecture and code were designed so I can evolve them together with AI assistants. The AI is my pair-programming buddy — it helped write the Rust, not the reverb. The plugin itself runs on pure, deterministic math. If it sounds bad, you can't blame the AI — that one's on you (or me). 🤖🔨
Q: Why CLAP only? Will there be VST3?
CLAP is the future — open, modern, no legacy cruft. I use Bitwig and Reaper myself. If your DAW can't load CLAP: switch to one that can, or send their support team persistently friendly emails until they wake up. 🦖✉️
Q: Are the plugins really free?
Yes — completely free. I've always supported open-source and freeware, and I'm paying it forward. If the plugins make your sessions better and you want to keep my caffeine levels healthy, sponsor me on GitHub or buy me a coffee ☕️🙏
Q: How do I install CLAP plugins?
Drop the .clap file into your CLAP folder. On Windows that's typically C:\Program Files\Common Files\CLAP. On Linux it's ~/.clap. Restart your DAW and it should pick them up automatically. No installer needed.
Q: Do the plugins work on Windows 10?
Audio processing works. Static controls — knobs, sliders, buttons — all work normally on Windows 10. However, the animated displays (spectrum analyzer, goniometer, peak meters) will appear blank. These real-time visualizations rely on a GPU-accelerated rendering pipeline (Iced + wgpu) that uses Windows 11's modern window-manager integration. On Windows 10, the graphics framework cannot reliably detect when the plugin window is ready to receive animated frames, so it never starts drawing them.

This is not a GPU hardware issue — any DirectX 12 capable GPU works fine. It's a window-lifecycle mismatch between the Iced GUI toolkit and Windows 10's compositor. Since Windows 10 is end-of-life, we target Windows 11 as the minimum supported platform for the full experience.
Q: What is SNAP and how do I use it?
SNAP exports your complete plugin state — spectrum data, band levels, all parameters — as a structured markdown summary. Copy it from the plugin UI and paste it into ChatGPT, Claude, or any LLM. You'll get mix analysis, frequency balance suggestions, and troubleshooting tips — all from an AI that sees exactly what your plugin sees, without ever touching the audio itself.
